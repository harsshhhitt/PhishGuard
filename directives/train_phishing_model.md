## Goal
Train a Random Forest classifier using the clean feature dataset generated by `extract_url_features.py`, evaluate its performance, and save the artifacts for later inference.

## Steps

1. **Ensure prerequisites:**
   - The feature dataset must exist (e.g. `.tmp/features.csv`) meaning `extract_url_features.py` has been run.
   - Install dependencies:
     ```powershell
     pip install scikit-learn pandas joblib
     ```

2. **Run the script:**
   ```powershell
   py execution/train_phishing_model.py
   ```
   *(By default, this looks for `.tmp/features.csv` and saves to `.tmp/`)*
   
   Or specify explicit paths:
   ```powershell
   py execution/train_phishing_model.py --input custom_data/my_features.csv --outdir custom_data/
   ```

3. **Verify Output:**
   - Observe the metrics printed to stdout (Accuracy, Precision, Recall, F1).
   - Check that `model.pkl` and `features.json` operate safely in the designated output folder.

## Edge cases & notes
- **Missing Dataset:** If the input CSV is not found, the script gracefully logs a clear error message and exits (code 1), instead of crashing.
- **Label Formatting:** The script heuristically handles object/string labels (like `"phishing"` vs `"legitimate"`) or numeric labels mapping them to a binary `[0, 1]` target for scikit-learn.
- **Missing Values:** NaNs remaining from malformed URLs in the feature extraction stage are filled with `0` before training, to prevent the classifier from throwing `ValueError`.
